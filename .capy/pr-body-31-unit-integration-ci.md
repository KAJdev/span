This PR adds a baseline test suite and CI checks to enable branch protection on `main`.

Summary of changes
- Unit tests: Added focused tests for event subject routing and log buffering/broadcasting behavior.
- Integration tests: Existing control-plane integration tests kept; CI runs them in a dedicated workflow with Docker available for `testcontainers`.
- CI workflows: Two separate GitHub Actions workflows (`unit-tests` and `integration-tests`) so branch protection can require both checks.

Nature of changes
- Tests and CI only; no production logic changed.

Impact
- Reliable, fast unit test job for quick feedback.
- Separate, slightly heavier integration test job validating NATS pub/sub and websocket streaming paths.

Checks introduced
- unit-tests — runs `cargo test --workspace --lib --bins`
- integration-tests — runs `cargo test --workspace --tests`

Notes
- Integration tests rely on Docker (testcontainers); the workflow verifies Docker availability on `ubuntu-latest`.
